    Replace the entire ProfileAnalyzerOutput model with the correct one above.
    Also change _output_key() in ProfileAnalyzer.py from "profile_analysis"
    to the INDIVIDUAL FIELDS, OR make BaseWorkerAgent unpack the output —
    see Section 3.1 for the recommended approach.

4.1  invoke() writes Pydantic model OBJECT to state, not individual fields
  CURRENT:
    return {
        **state,
        self._output_key(): response,   ← response is a ProfileAnalyzerOutput OBJECT
        self._done_key(): True,
    }

  PROBLEM FOR ProfileAnalyzer:
    - _output_key() = "profile_analysis"
    - But RoadmapState has NO "profile_analysis" key
    - Supervisor checks state["difficulty_level"], state["skill_gaps"] directly
    - These are NEVER written → supervisor always falls back incorrectly

  FIX OPTION A (recommended — clean):
    In ProfileAnalyzer, override _write_to_state() hook:
    def _write_to_state(self, response, state):
        return {
            **state,
            "difficulty_level": response.difficulty_level,
            "skill_gaps": response.skill_gaps,
            "prerequisite_analysis": response.prerequisite_analysis,
            "estimated_duration_weeks": response.estimated_duration_weeks,
            "profile_agent_done": True,
        }

    Then in BaseWorkerAgent.invoke(), call self._write_to_state(response, state)
    instead of the hardcoded dict.

  FIX OPTION B (simpler, all agents): Add model_dump() call in invoke():
    return {
        **state,
        **response.model_dump(),   ← spread all Pydantic fields directly into state
        self._done_key(): True,
    }
    This works if the Pydantic field names EXACTLY match the RoadmapState keys.

  RECOMMENDATION: Use Option B for ProfileAnalyzer once schema is fixed (Section 1.1).
    For other agents (ResourceCurator, AdaptationEngine), keep _output_key()
    approach since they write to a single key ("resources", "adapted_curriculum").
    But also fix ResourceCurator to write to "stage_resources" not "resources"
    during stage progression mode (see Section 4.2).
